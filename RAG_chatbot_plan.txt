THÔNG TIN QUERY từ bảng `movies` JOIN `showtimes` JOIN `rooms` JOIN `theaters`:
  - movie_id, title, genre, description, director, language, duration
  - status (now_showing / coming_soon), imdb_rating
  - start_time, end_time, format (2D/3D/IMAX)
  - room_name, theater_name, theater_address

MỖI BẢN GHI PHIM được chuyển thành LangChain Document với:
  - page_content : Chuỗi văn bản mô tả phim (sẽ được embedding)
  - metadata     : Dict chứa movie_id, poster_url, status, genre (dùng để filter)

LÝ DO CHỌN CÁCH NÀY:
  - Dữ liệu luôn mới (real-time từ DB) thay vì file JSON cứng
  - Gộp suất chiếu vào cùng document giúp AI biết "phim này chiếu lúc nào, ở đâu"
  - Trường "Tâm trạng phù hợp" được sinh tự động từ genre để tăng độ chính xác
    khi user hỏi theo cảm xúc thay vì tên phim

CODE (ai_service.py):
  async def load_movie_documents(db: AsyncSession) -> List[Document]:
      stmt = (
          select(
              MovieTable,
              func.string_agg(
                  ShowtimeTable.start_time.cast(Text) + " tại " +
                  TheaterTable.name + " (" + RoomTable.name + "-" + ShowtimeTable.format + ")",
                  ", "
              ).label("showtime_info")
          )
          .outerjoin(ShowtimeTable, ShowtimeTable.movie_id == MovieTable.movie_id)
          .outerjoin(RoomTable, RoomTable.room_id == ShowtimeTable.room_id)
          .outerjoin(TheaterTable, TheaterTable.theater_id == RoomTable.theater_id)
          .where(MovieTable.status == "now_showing")  # Chỉ lấy phim đang chiếu
          .group_by(MovieTable.movie_id)
      )
      result = await db.execute(stmt)
      rows = result.all()

      documents = []
      for movie, showtime_info in rows:
          mood_tags = genre_to_mood(movie.genre)  # Hàm map genre -> tâm trạng
          content = f"""
  Tên phim: {movie.title}
  Thể loại: {movie.genre}
  Đạo diễn: {movie.director}
  Ngôn ngữ: {movie.language}
  Thời lượng: {movie.duration} phút
  IMDb: {movie.imdb_rating or 'Chưa có'}/10
  Trạng thái: {'Đang chiếu' if movie.status == 'now_showing' else 'Sắp chiếu'}
  Nội dung: {movie.description}
  Suất chiếu: {showtime_info or 'Chưa có lịch chiếu'}
  Tâm trạng phù hợp: {mood_tags}
          """.strip()

          doc = Document(
              page_content=content,
              metadata={
                  "movie_id": movie.movie_id,
                  "title": movie.title,
                  "genre": movie.genre,
                  "poster_url": movie.poster_url or "",
                  "status": movie.status,
                  "imdb_rating": movie.imdb_rating or 0,
              }
          )
          documents.append(doc)
      return documents

HÀM genre_to_mood (ánh xạ thể loại → từ khóa tâm trạng):
  def genre_to_mood(genre: str) -> str:
      mapping = {
          "Hành động": "hào hứng, kịch tính, mạo hiểm, phấn khích",
          "Hài":       "vui, cười, chill, thư giãn, nhẹ nhàng",
          "Tình cảm":  "lãng mạn, nhớ người yêu, muốn khóc, cảm xúc",
          "Kinh dị":   "hồi hộp, sợ, mạo hiểm, kịch tính",
          "Hoạt hình": "vui, gia đình, trẻ em, chill",
          "Gia đình":  "ấm áp, gia đình, vui, cảm xúc",
          "Tâm lý":    "suy nghĩ, triết lý, trầm, tĩnh tâm",
          "Khoa học viễn tưởng": "mạo hiểm, kỳ diệu, sáng tạo",
          "Chiến tranh": "hào hùng, cảm động, kịch tính",
          "Tài liệu":  "học hỏi, tĩnh tâm, suy nghĩ",
      }
      tags = []
      for key, val in mapping.items():
          if key.lower() in genre.lower():
              tags.append(val)
      return ", ".join(tags) if tags else "giải trí chung"

QUYẾT ĐỊNH: KHÔNG chunking (1 Document = 1 phim)

LÝ DO:
  - Mỗi phim đã là một đơn vị thông tin độc lập, khoảng 200–400 tokens
  - Nếu chunk nhỏ hơn, có thể tách rời "tên phim" khỏi "suất chiếu",
    khiến AI không biết phim đó chiếu lúc mấy giờ
  - Với ~20 phim (quy mô demo), tổng document nhỏ, không cần chunk thêm

NẾU SAU NÀY CÓ MÔ TẢ PHIM DÀI (>800 tokens):
  from langchain.text_splitter import RecursiveCharacterTextSplitter

  splitter = RecursiveCharacterTextSplitter(
      chunk_size=500,          # ~500 tokens mỗi chunk
      chunk_overlap=50,        # 50 token overlap để không mất ngữ cảnh ranh giới
      separators=["\n\n", "\n", ". ", " "]  # Ưu tiên tách theo đoạn văn
  )
  chunks = splitter.split_documents(documents)

HIỆN TẠI: Bỏ qua bước chunking, dùng thẳng documents từ Bước 1.

TẠI SAO KHÔNG DÙNG openai/google embedding:

    đặc biệt hỗ trợ đa ngôn ngữ bao gồm tiếng Việt

BƯỚC 4: VECTOR DATABASE – FAISS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CHỌN FAISS: Facebook AI Similarity Search
  - Không cần server riêng (khác ChromaDB server mode)
  - Chạy trong RAM của FastAPI process
  - Phù hợp với ~20-100 phim (quy mô demo)
  - Tìm kiếm vector cực nhanh (microseconds)

CẤU TRÚC LƯU TRỮ:
  backend/
  └── faiss_index/
      ├── index.faiss     ← Vector index (binary file)
      └── index.pkl       ← Document store + metadata

CHIẾN LƯỢC REBUILD INDEX:
  - Index được build khi server khởi động (hoặc khi admin trigger rebuild)
  - Tự động rebuild sau 30 phút (background task) để cập nhật phim mới/suất mới
  - Dùng biến global `vectorstore` trong memory để tránh rebuild mỗi request

CODE – Build FAISS Index:
  from langchain_community.vectorstores import FAISS
  import os, pickle

  FAISS_INDEX_PATH = "faiss_index"

  async def build_or_load_vectorstore(db: AsyncSession) -> FAISS:
      """
      Nếu đã có index trên disk → load lại (nhanh).
      Nếu chưa có → build từ DB (chậm hơn, ~5-10 giây).
      """
      if os.path.exists(f"{FAISS_INDEX_PATH}/index.faiss"):
          # Load index đã có
          vectorstore = FAISS.load_local(
              FAISS_INDEX_PATH,
              embedding_model,
              allow_dangerous_deserialization=True
          )
      else:
          # Build mới từ DB
          documents = await load_movie_documents(db)
          vectorstore = FAISS.from_documents(documents, embedding_model)
          vectorstore.save_local(FAISS_INDEX_PATH)  # Lưu xuống disk

      return vectorstore

CODE – Rebuild Index (dùng cho background task hoặc admin API):
  async def rebuild_vectorstore(db: AsyncSession) -> FAISS:
      documents = await load_movie_documents(db)
      vectorstore = FAISS.from_documents(documents, embedding_model)
      vectorstore.save_local(FAISS_INDEX_PATH)
      return vectorstore

BƯỚC 5: RETRIEVER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LOẠI RETRIEVER: Similarity Search (cosine similarity)

NGUYÊN LÝ:
  1. Câu hỏi của user được embed thành vector (dùng nomic-embed-text)
  2. So sánh cosine similarity giữa query vector và tất cả document vectors trong FAISS
  3. Trả về top-k document có độ tương đồng cao nhất

VÍ DỤ:
  User hỏi: "Tôi đang buồn, muốn xem phim gì?"
  → Embed câu hỏi → vector [0.12, -0.34, 0.89, ...]
  → So sánh với tất cả document vectors
  → Document "Inside Out 2" (có "tâm trạng phù hợp: cảm xúc, gia đình") có
    cosine similarity = 0.87 → được retrieve
  → Document "Fast & Furious 11" (hành động) cosine similarity = 0.41 → không được retrieve

CODE:
  retriever = vectorstore.as_retriever(
      search_type="similarity",    # Dùng cosine similarity (mặc định với FAISS)
      search_kwargs={
          "k": 4                   # Lấy top 4 phim phù hợp nhất
      }
  )

TẠI SAO k=4?
  - Quá ít (k=1): AI chỉ thấy 1 phim → gợi ý nghèo nàn
  - Quá nhiều (k=8): prompt quá dài → LLM bị "overwhelmed", chậm
  - k=4: cân bằng – đủ để AI có lựa chọn gợi ý, prompt vừa phải

NÂNG CẤP NẾU CẦN (MMR – Maximum Marginal Relevance):
  Tránh trả về 4 phim giống nhau cùng thể loại:
  retriever = vectorstore.as_retriever(
      search_type="mmr",
      search_kwargs={"k": 4, "fetch_k": 10, "lambda_mult": 0.7}
  )
  # lambda_mult: 1.0 = chỉ quan tâm relevance, 0.0 = chỉ quan tâm diversity

  Giải thích thang temperature:
  ┌───────────────────────────────────────────────────────────────────────┐
  │  0.0 → Deterministic: Luôn trả lời giống nhau, máy móc, ít sáng tạo  │

  └───────────────────────────────────────────────────────────────────────┘

  - Đủ sáng tạo để viết câu trả lời tự nhiên, không nhàm chán
  - Không quá cao để AI bịa tên phim không tồn tại
  - Phù hợp với tone thân thiện của chatbot tư vấn

BƯỚC 8: RAG CHAIN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DÙNG LCEL (LangChain Expression Language) – Cú pháp pipe (|)

LUỒNG XỬ LÝ:
  Input: { question, chat_history }
      │
      ├─► retriever.invoke(question)  → List[Document]  (top 4 phim)
      │   │
      │   └─► format_docs(docs)       → context string (nối page_content)
      │
      ▼
  prompt.format(context, chat_history, question)
      │
      ▼
  llm.invoke(prompt_value)
      │
      ▼
  StrOutputParser()                   → String phản hồi thuần túy

CODE HOÀN CHỈNH (ai_service.py):
  from langchain_core.output_parsers import StrOutputParser
  from langchain_core.runnables import RunnableParallel, RunnablePassthrough
  from langchain_core.messages import HumanMessage, AIMessage

  def format_docs(docs: list) -> str:
      """Nối nội dung các Document thành 1 chuỗi context"""
      return "\n\n---\n\n".join([doc.page_content for doc in docs])

  def build_rag_chain(vectorstore: FAISS):
      retriever = vectorstore.as_retriever(
          search_type="similarity",
          search_kwargs={"k": 4}
      )

      # LCEL Chain
      rag_chain = (
          RunnableParallel({
              "context":      retriever | format_docs,  # Retrieve → format
              "question":     RunnablePassthrough(),     # Pass question thẳng
              "chat_history": RunnablePassthrough(),     # Pass history thẳng
          })
          | prompt         # Điền vào template
          | llm            # Gọi Ollama
          | StrOutputParser()  # Parse output thành string
      )
      return rag_chain

CÁCH GỌI CHAIN:
  from langchain_core.messages import HumanMessage, AIMessage

  async def chat(
      question: str,
      history: list[dict],     # [{"role": "user", "content": "..."}, ...]
      vectorstore: FAISS,
      db: AsyncSession
  ) -> dict:

      # Convert history sang LangChain message format
      chat_history = []
      for msg in history:
          if msg["role"] == "user":
              chat_history.append(HumanMessage(content=msg["content"]))
          elif msg["role"] == "assistant":
              chat_history.append(AIMessage(content=msg["content"]))

      # Build chain và invoke
      chain = build_rag_chain(vectorstore)
      reply = await chain.ainvoke({
          "question": question,
          "chat_history": chat_history,
      })

      # Lấy metadata của các phim được retrieve để trả suggested_movies
      retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
      retrieved_docs = retriever.invoke(question)
      suggested_movies = [
          {
              "movie_id":   doc.metadata["movie_id"],
              "title":      doc.metadata["title"],
              "poster_url": doc.metadata["poster_url"],
              "genre":      doc.metadata["genre"],
          }
          for doc in retrieved_docs
          if doc.metadata.get("status") == "now_showing"
      ]

      return {
          "reply": reply,
          "suggested_movies": suggested_movies
      }

  200 OK:
  {
      "reply": "Bạn đang cần chút vui vẻ hả? Mình gợi ý bạn...",
      "suggested_movies": [
          {
              "movie_id":   5,
              "title":      "Inside Out 2",
              "poster_url": "https://...",
              "genre":      "Hoạt hình, Gia đình"
          },
          {
              "movie_id":   12,
              "title":      "Deadpool & Wolverine",
              "poster_url": "https://...",
              "genre":      "Hành động, Hài"
          }
      ],
      "conversation_id": 42      ← ID phiên chat (null nếu Guest)

API #2 – GET /ai/conversations  (CONVERSATION LIST)
──────────────────────────────────────────────────────────────────────────────
  Mục đích : Lấy toàn bộ phiên chat của user đang đăng nhập
  Auth     : JWT Required (lấy user_id từ token)

  RESPONSE:
  200 OK:
  {
      "conversations": [
          {
              "conversation_id": 42,
              "started_at":       "2026-06-14T10:30:00Z",
              "preview":          "Tôi đang buồn, muốn xem phim gì...",
              "message_count":    6
          },
          {
              "conversation_id": 38,
              "started_at":       "2026-06-12T15:00:00Z",
              "preview":          "Phim hành động nào hay nhất?",
              "message_count":    4
          }
      ]
  }
  → preview: Nội dung tin nhắn đầu tiên của user trong phiên (để user nhận ra)
  → Sắp xếp theo started_at DESC (phiên mới nhất lên trước)

  DB QUERY:
  SELECT c.conversation_id, c.started_at,
         (SELECT content FROM ai_messages
          WHERE conversation_id = c.conversation_id
            AND sender_type = 'user'
          ORDER BY sent_at LIMIT 1) AS preview,
         COUNT(m.message_id) AS message_count
  FROM ai_conversations c
  LEFT JOIN ai_messages m ON m.conversation_id = c.conversation_id
  WHERE c.customer_id = <user_id>
  GROUP BY c.conversation_id
  ORDER BY c.started_at DESC;
